# 《无人机跨域认证》方案

## 1.BACKGROUND

​	本方案着眼于无人机跨域认证，看起来很不错，能够解决无人机在长途飞行途中的跨管理域新人问题，但此时就会有一些困惑——无人机这种飞行距离一般不会很远的高功能飞行器还会需要跨域这种听着就很“远”的功能吗？这种功能的实用性到底如何呢？这里我们并不做定量的分析，毕竟也没那么多数据不是，只做定性的分析。

​	首先对于无人机这一具体对象而言，我们应该明确的是现在消费级无人机的飞行性能已经很可观了，每年都有数不胜数的“黑飞”事件，而且随着国力不断增长，“黑飞”的无人机数量也不断在增长。所谓“黑飞”，顾名思义，脱离原来的管理域，未经许可飞入其他空域，通常情况下，这类无人机要么伪装合法身份，要么干脆就不去与目标域通信。我们设想的方案首先是建立在无人机无论如何都需要建立与目标域的新人的这一基础上，在保障合法无人机成功跨域的同时，精准捕获“黑飞”的无人机，因此，方案主要针对的对象是前者（i.e.,伪装身份以获取目标域信任的非法无人机）。

​	其次，是关于域的界定。事实上，在我国目前的空域管理制度下，合法飞行的无人机只要提前与有关部门申请，就可以在目标空域的指定范围内自由飞行，即便确实需要跨越管理域进行飞行任务，也会有或正式、或临时的第三方证明去获取飞行许可。那么，在上述语境中，“域”的划分就是完全由相关部门划定的。我们方案的任务就是保障跨域飞行认证的安全、高效、可靠。但具体而言，想要保证高效的话，就要对域的边界做一点小小的预处理——弹性内边界，i.e.,在域的边界处处处正交地往内缩个若干距离，定为【出域界限】。无人机只要越过这条线并在限定时间未返回最內域，我们就认为无人机想要出域，此时再去检测相邻域的【入域界限】，若无人机同时处于两者间，就向该域发送事前认证的请求。正式进入域后，正式认证，然后执行一些其他的更新机制啥的。

## 3.PRELIMINARIES

### 3.1 Bilinear Mapping

​	简而言之，一个双线性映射是指从两个加法循环群的笛卡尔积映射到乘法循环群，并保持同态的这么一种映射，i.e., $e: G_1\times G_2\longrightarrow G_T，满足e(ka,b)=e(a,kb)=e(a,b)^k，其中k\in \mathbb{R}，同时e(P,Q)\neq 1,这里的P、Q分别是G_1、G_2的生成元$，简而言之，要同时满足双线性性和非退化性。不过这样的定义还不足以作为完整的双线性映射，想要将其运用到密码学中去，可计算是必须的，对于一个可以实际应用的双线性对，应当有一个高效的多项式时间复杂度算法来计算$e(a,b)$的值,在实际应用中一般使用**Weil Pairing**或者**Tate Pairing**及其变体，e.g., Boneh-Lynn-Shacham(BLS)常用的商群构造。

​	从上面的定义出发，可以很自然地利用指数运算的复杂性以及同态的良好性质，定义出性质非常好的数字签名方案，能够充分保护隐私。

​	不过说的是天花乱坠的，双线性对对比传统ECC开销很大、密钥和密文长度膨胀、“实现极度复杂、难以标准化”，并且由于其基于离散对数问题，而在映射后的$G_T$中具体为有限域上的传统离散对数问题，此时若是$G_T$的特征值过小，则已有数学家提出了针对小特征值有限域上离散对数问题的准多项式时间算法，以及侧信道攻击、单点故障、难以抗量子等特性，实际上若是整个方案都是用双线性对构造是不现实的，效率极低。因此，在方案设想中，双线性对只会作为数字签名的一个组件出现。

### 3.3 Extended Chebyshev Chaotic Map(ECCM)

​	关于这个映射关系呢，是我在一篇论文中看到的，在那篇论文中作者称ECCM具有良好且完备的安全性保证，具体而言，利用浮点数和超越函数的良好性质构造非线性动力学方程以生成伪随机数，但这会有一个小问题，就是计算机内的浮点数并不直接等同于数学上的实数，而是会有溢出的问题，这会打乱ECCM的伪随机特性，且相较于使用更广泛的ECC(Elliptic Curve Cryptography)，ECCM最大的问题在于其及其高昂的浮点计算成本。

### 3.4 Lattice

​	格的定义很多，简单来讲就是一个高维的线性变换，在目标加噪的情况下，寻找最合适的整向量。其核心算法极高的难度，使得基于格的密码学构造往往可以认为是能够抗量子的，也是后量子密码学的代表。

​	格的构造大体可以认为是高维实线性空间中线性无关向量组的全体整系数线性组合，密码学中得到应用的格上的难题可列表如下：

| 难题                       | 内容                                                         |
| -------------------------- | ------------------------------------------------------------ |
| 最短向量问题（SVP）        | 给定一个格的基，找出该格中长度最短的向量；此问题由于可能的坏基的存在，因而即使是量子计算机，想要在指数级复杂的都多空间中找到最短的向量也是难如登天 |
| 最近向量问题（CVP）        | 给定一个不在格上的任意空间点$t$，在格中找出距离$t$最近的那个格点 |
| 容错学习问题（LWE）        | 求解一个带有误差的线性方程组，形如$Ax+e=b$，对此量子计算机也无奈了。分为搜索型和判定型两类。 |
| 环上的容错学习问题（RLWE） | 克服了标准LWE数据传输量太大的问题，现在只需要传输系数即可。  |

​	事实上，由于格上难题的抗量子性，基于格的密码学也是现代抗量子密码体系的代表之一，其计算速度极快，但属于安全性考量，相应的密钥往往需要ECC的数倍乃至数十倍。

​	当下的抗量子密码除了基于格的密码学之外，还有基于哈希的密码学，两者的共同点就是逆向的计算复杂性过高，以至于量子计算机也无用，从而实现抗量子的目的。

### 3.5 Proxy Re-Encryption

​	在不进行解密的时候，直接在内部用第二个密钥进行加密，并同时解开第一个密钥的加密。为什么会有这种技术的出现，顾名思义，这是在客户端对远程代理的某个过程进行的，数据并不在本地，但需要用到本地的密钥，而本地并不想给服务端透露自己的密钥。代理重加密实际上是一个很笼统的叫法，可以认为是多种技术的统称，其中最符合概念的应该是全同态加密，在服务端上自始至终都只能看到密文在不断变化，而服务端完全没有能力解开密文。

​	具体而言，经典的代理重加密方案有基于双线性对乘法同态特性的重加密方案、格密码, etc.

### 3.6 方案的评判标准(安全性、效率、通信开销)

​	**安全性：**就是针对当前方案，评估其是否足够可靠，是否会存在已知的安全风险，这个性质可以参考其他论文的评判标准：

| Security Property          | Detailed Contents of the certain property                    |
| -------------------------- | ------------------------------------------------------------ |
| Mutual Authentication      | 目标域需要确定无人机不是“黑飞”，无人机也需要确定目标域来接洽的通信节点是真实的 |
| Unforgeability             | 认证的双方在对方接收到消息后，若消息合规，则都不可否认消息的来源 |
| Anonymity                  | 系统内部各实体间通信时，并不使用真实身份，这样一来，通信双方都将不能获取对方的真实身份 |
| Unlinkability              | 攻击者即使获取了系统中的多次操作记录、数据或行为，也无法推断出这些记录属于同一个主体（用户、无人机或设备） |
| Location/Geofence Privacy  | 顾名思义，为无人机的地理位置添加隐私保护                     |
| Forward/Backward Security  | 前向安全，即授权失效前消息的安全性；后向安全，即授权失效后消息的安全性 |
| Physical Tamper-Resistance | 即使无人机被物理捕获，也不会泄露隐私或秘密                   |
| Anti-Replay & MITM         | 防止攻击者仅仅重放消息便可冒充合法身份的情况发生             |

​	**效率：**方案运行时各模块的运行效率，具体而言就是各个相对分离的阶段中，数据处理的速度如何。

​	**通信开销：**在实体间通信时，所需要传输的包的大小，通常以bit为单位。
